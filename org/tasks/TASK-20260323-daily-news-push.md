# 每日新闻推送系统

## 上下文

主人 Graves 在 News 群（oc_93086ae445f1b1f08c955acb931800b7）提出需求：
- 每天早上 8:30 在 News 群推送新闻
- 内容包括：
  1. 加密货币的投资意见和最新动态
  2. AI 圈的最新动态
  3. OpenClaw 的最新动态
  4. GitHub 最火的五个项目和介绍

## 阶段1 - 需求拆解（计府）

### 可交付项
1. **新闻聚合模块**：从多个信源获取新闻内容
   - 加密货币数据（CoinGecko/CoinMarketCap API）
   - AI 新闻（Google 搜索/技术博客）
   - OpenClaw 动态（GitHub releases/官方渠道）
   - GitHub 热门项目（GitHub Trending API）

2. **内容生成模块**：将原始数据转化为有可读性的新闻
   - 数据分析与摘要
   - 自然语言生成
   - 格式美化（飞书富文本）

3. **定时推送模块**：按计划执行推送
   - OpenClaw cron 配置
   - 飞书群消息发送
   - 失败重试与日志

4. **验收与迭代**：测试验证与优化

### 里程碑
- M1：新闻聚合模块完成（能获取各类数据）
- M2：内容生成模块完成（有可读性）
- M3：定时推送完成并验收通过

### 依赖

### 范围边界

## 阶段2 - 市场/价值/成本分析（计府）

### 市场分析
- **需求明确性**：高（主人直接提出，有具体内容要求）
- **复用价值**：中（可复用于其他群或频道）
- **替代方案**：手动搜索（耗时）、第三方推送（不定制）

### 价值分析
- **直接价值**：节省主人 10-15 分钟/天的新闻搜索时间
- **间接价值**：信息及时获取、投资决策参考、技术趋势跟踪
- **优先级**：高（主人明确需求）

### 成本分析
- **开发成本**：1-2 小时（已有基础框架，主要是内容质量提升）
- **运行成本**：低（API 调用免费或低费率）
- **维护成本**：低（每月检查一次信源是否变化）
- **Token 估算**：每次推送约 1000-2000 tokens，每月约 60000 tokens

### 不确定项
- 外部 API  rate limit（假设充足）
- 新闻源稳定性（假设主流信源稳定）
- 内容质量迭代次数（假设 1-2 次）
